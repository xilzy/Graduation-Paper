# store2hdf5.m

一个自定义函数，用于后面主函数生成h5文件

## <1>Input

###(1)filename:

```
文件路径和文件名，例如./xxx.h5
```

### (2)data:

```
归一化的W,H,C,N矩阵（W代表水平方向像素个数，H代表竖直反向像素个数，C代表通道数，N代表图像的张数）
```

这里补充一下两种格式的图像数据

![image-20220906212923799](C:\Users\HUAWEI\AppData\Roaming\Typora\typora-user-images\image-20220906212923799.png)

NCHW格式同一个通道内的像素存储在一起，RRRRGGGGBBBB，[batch_size,channel,height,width]

NHWC则是以图像空间位置为顺序，同一个位置的三通道数据存储在一起，RGBRGBRGBRGB [batch_size height width channel]

### (3)create:

```
确定创建方式是新建还是追加（默认为值为1，新建模式）
0:追加模式
1:新建模式
```

### (4)startloc:

```
数据起始写入位置指针：默认情况下
如果create=0,则startloc.data=[1 1 1 K+1],startloc.lab = [1 K+1]; 其中K是当前存入HDF文件中的样本数
如果create=1,则startloc.data=[1 1 1 1],startloc.lab=[1 1];
```

### (5)chunksz:

```
只在新建模式(create=1)下有效，指定每一块中样本数量的多少，块越大，读写操作越快
```

## <2>output

```
curr_dat_sz：返回当前数据集的大小
```

## <3>功能：

```
创建一个h5文件并写入内容，并返回h5文件中当前写入的数据集大小
```

## <4>实现功能的主要函数

### h5create

```
<1>Input:h5create(filename,ds,sz,Name,Value,……)
(1)filename:h5文件名（带路径）
(2)ds:数据集的名字（因为h5文件有组合数据，所以这里用字符串向量来区分组合数据，例如'/g1/g2/myDataset'就代表三组和一个数据集，三个组分别是/,g1,g2,数据集是myDataset）
(3)sz:数据集大小，行向量，例如[100 200 300]就是100x200x300的数据集，如果要指定无限维度,使用Inf指定(要使用Inf，必须设置ChunkSize参数)
(4)Name,value:matlap中的通过键值对指定参数赋值，常见的如下
'Datatype':'double'(默认)'single''uint64''int64''uint32''int32''uint16''int16''uint8''int8''string'
'ChunkSize':行向量指定，行向量维度必须和sz一致

<2>Output:无输出

<3>功能：创建.h5类型文件
```

### h5write

```
<1>Input:h5write(filename,ds,data,start,count)
(1)(2)前两个参数和h5create一模一样
(3)data:数值矩阵，要写入h5文件的数据，类型必须和创建数据集时候的一致。如果不指定 start 或 count，则数据必须为非稀疏的，并且必须与 HDF5 数据集大小相同。如果数据集中的一个维度是无限的，则要写入的数据的该维度可以是任何大小。
(4)start:数值向量，向量中的每一个元素指定对应维度的起始索引位置，未指定默认从1开始 
(5)count:数值向量，向量中的每一个元素指定对应维度写入的数据的个数，默认是Inf向量

<2>Output:无输出

<3>功能:像h5文件中写入内容
```

### h5info

```
<1>Input:info = h5info(filename)
filename:h5的文件名

<2>Output:一个信息结构体

<3>功能:返回整个 HDF5 文件的有关信息，包括其中包含的组、数据集和命名数据类型的有关信息
```

## stride

这个变量的中文意思是步长，然后相关代码出现在子图像制作的地方，但是我调整步长之后，并没有发现子图像有什么改变，然后最后制作出来的h5文件大小也没有改变，所以推测可能步长的大小影响子图像制作的速度吧，相关代码如下(纠正，步长决定采样频率，但是要搭配input_size才有效)

```matlab
   for x=1:stride:(height-size_input+1)
        for y=1:stride:(width-size_input+1)    
            sub_img1(:,:,:)=i1(x:x+size_input-1,y:y+size_input-1,:);
            sub_img2(:,:,:)=i2(x:x+size_input-1,y:y+size_input-1,:);
            count=count+1;
            data(:,:,1:1+img_ch-1,count)=sub_img1(:,:,:);
            data(:,:,1+img_ch:1+2*img_ch-1,count)=sub_img2(:,:,:);
              
            %这里有画图显示
            figure(1)
            A=data(:,:,1:1+img_ch-1,count);
            B=data(:,:,1+img_ch:1+2*img_ch-1,count);
            subplot(221),imshow(A);
            subplot(222),imshow(B);
            subplot(223),imshow(sub_img1);
            subplot(224),imshow(sub_img2);
        end
    end
```

# 最重要的几个参数

```
<1>chuncksz对应batch_size:即每一批多少块
<2>input_size对应patch_size：即每一块多大
<3>图片的height和width 和stride以及input_size共同决定一张图片能制作多少块
num=int[(height-inputsize)/stride]*int[(width-input_size)/stride]
<4>如果再考虑了放大比例ratio之后
num=int[(height*ratio-inputsize)/stride]*int[(width*ratio-input_size)/stride]
例如256x256原图片，stride=64,ratio=4,input_size=256
则patch_num//picture_num=int[(256*4-256)/64]*int[(256*4-256)/64]=144块
```
